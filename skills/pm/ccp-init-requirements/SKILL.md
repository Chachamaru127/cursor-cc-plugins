# Init Requirements

新しいプロジェクトやタスクの要件をヒアリングし、構造化するスキル。
VibeCoder が「〇〇を作りたい」と言うだけで、実行可能な計画を生成します。

---

## 入力

- **ユーザーの要望**: 自然言語での説明（例: 「ブログを作りたい」）
- **コンテキスト**: 既存のプロジェクト情報（あれば）

---

## 出力

- **構造化された要件定義**
- **技術スタックの提案**
- **Plans.md への初期タスク追加**

---

## 実行手順

### Step 1: やりたいことをヒアリング

> 🎯 **どんなものを作りたいですか？**
>
> ざっくりで大丈夫です！例えば：
> - 「ブログ」「ポートフォリオ」「ECサイト」
> - 「タスク管理アプリ」「チャットアプリ」
> - 「社内ツール」「ダッシュボード」
>
> 思いついたまま教えてください 👇

**回答を待つ**

### Step 2: 解像度を上げる質問（2-3問）

> 📋 **もう少し教えてください：**
>
> 1. **誰が使いますか？**（自分だけ / チーム / 一般公開）
> 2. **似てるサービスはありますか？**（例: Notion, Zenn, メルカリ）

**回答を待つ**

### Step 3: 技術スタックの決定

`cursor-cc.config.json` の `tech_choice_mode` に従う：

| モード | 動作 |
|--------|------|
| `fixed` | 設定された `base_stack` を使用 |
| `ask` | 選択肢を提示してユーザーに確認 |
| `auto` | AI が最適な技術を選択して報告 |

### Step 4: 要件の構造化

```markdown
## 🎯 プロジェクト: {{プロジェクト名}}

### 概要
- **目的**: {{ユーザーのやりたいこと}}
- **対象**: {{誰が使うか}}
- **参考**: {{類似サービス}}

### 技術スタック（提案）
- フロントエンド: {{推奨技術}}
- バックエンド: {{推奨技術}}
- データベース: {{推奨技術}}
- デプロイ: {{推奨サービス}}
```

### Step 5: Plans.md へのタスク追加

```markdown
## 🔴 フェーズ1: 基盤構築 `cc:TODO`

- [ ] プロジェクト初期化 `cc:TODO`
- [ ] 基本ディレクトリ構造の作成 `cc:TODO`
- [ ] 開発環境のセットアップ `cc:TODO`
- [ ] Git 初期化 & 初回コミット `cc:TODO`

## 🟡 フェーズ2: コア機能 `cc:TODO`

- [ ] {{コア機能1}} `cc:TODO`
- [ ] {{コア機能2}} `cc:TODO`
- [ ] {{コア機能3}} `cc:TODO`

## 🟢 フェーズ3: 仕上げ `cc:TODO`

- [ ] UI の調整 `cc:TODO`
- [ ] テストの追加 `cc:TODO`
- [ ] デプロイ設定 `cc:TODO`
```

---

## VibeCoder 対応

- **専門用語は使わない**: または平易な説明を添える
- **選択肢は3つまで**: 迷わせない
- **デフォルトを用意**: 「おまかせ」で進められるように

---

## 出力後のアクション提案

> ✅ **計画が完成しました！**
>
> 📄 `Plans.md` に {{タスク数}} 個のタスクを追加しました。
>
> **次にやること：**
> 「フェーズ1を始めて」または「プロジェクトを作って」と言ってください。
